# 改动日志

## 2026-04-28 企业级增强（第1次追加）

### 本次改动目的

在已有 Agentic RAG 能力基础上，补齐企业开发最小闭环：**安全访问控制、请求可追踪、可部署、可自动验证**，降低上线风险并提升后续迭代可维护性。

### 本次改动内容

1. **接入 API Key 鉴权**

2. **增加请求追踪与访问日志**
   - 新增 `app/middleware.py`
   - 为每个请求生成/透传 `X-Request-Id`

3. **配置中心增强**
   - 扩展 `app/config.py`，新增：
     - `APP_ENV`
     - `API_KEY`
     - `ENABLE_AUTH`
     - `CORS_ALLOW_ORIGINS`

4. **可部署能力**
   - 新增 `Dockerfile`，支持标准容器启动 FastAPI 服务

5. **CI 与基础测试骨架**
   - 新增 `.github/workflows/ci.yml`
   - 新增 `tests/test_api_basics.py`
   - 增加健康检查与鉴权拦截的基础测试

6. **文档更新**
   - 更新 `README.md`：新增鉴权请求头说明、Docker 启动说明、企业级下一步建议
   - 更新 `.env.example`：补充企业级配置项

## 2026-04-28 大批量入库超时优化（第6次追加）

### 本次改动目的

针对异步任务 `stage=failed` 且 `504 Vector build timed out` 的场景，降低单次 embedding 请求负载，提升大规模文档入库成功率。

### 本次改动内容

1. **分批向量化写入（核心优化）**
   - 更新 `app/rag/vectorstore.py`
   - Chroma 从一次性 `from_documents` 改为按批 `add_texts`
   - FAISS 改为首批初始化 + 后续 `add_documents` 增量写入
   - 避免超大请求导致 Ollama embedding 超时

2. **新增可调批大小配置**
   - 更新 `app/config.py`
   - 新增 `ingest_batch_size`（默认 32）

3. **配置与文档同步**
   - 更新 `.env.example`：新增 `INGEST_BATCH_SIZE=32`
   - 更新 `README.md`：补充 `INGEST_BATCH_SIZE` 说明（建议 16~64）

## 2026-04-28 逐文件入库与低配提速（第7次追加）

### 本次改动目的

将异步导入改为逐文件串行处理，降低大批量入库时内存/请求峰值；同时针对低配设备优化问答链路，减少本地推理耗时。

### 本次改动内容

1. **入库流程切换为逐文件串行**
   - 更新 `app/main.py`
   - `_ingest_docs(...)` 从“全量加载+统一切分+统一建库”改为“按文件循环：加载 -> 切分 -> 写入向量库”
   - 异步任务进度细化为按文件推进（`Processed x/y`）

2. **新增文件级加载能力**
   - 更新 `app/rag/loader.py`
   - 新增 `list_supported_files(...)` 与 `load_document_file(...)`
   - 支持先扫描可处理文件，再逐个读取（`.pdf/.docx/.doc`）

3. **新增向量库增量写入能力**
   - 更新 `app/rag/vectorstore.py`
   - 新增 `create_incremental_writer(...)`
   - `Chroma/FAISS` 均支持逐批添加并最终收敛持久化

4. **低配设备默认加速参数**
   - 更新 `app/schemas.py`
   - `AskRequest` 默认值调整为：`top_k=2`、`max_iterations=1`

5. **问答快速模式与可调上下文**
   - 更新 `app/config.py`
   - 新增配置：
     - `RAG_FAST_MODE`（默认 `true`）
     - `RAG_RELEVANCE_MAX_CHARS`（默认 `2500`）
     - `RAG_ANSWER_MAX_CHARS`（默认 `4500`）
   - 更新 `app/rag/agentic_graph.py`
   - `RAG_FAST_MODE=true` 时采用 `retrieve -> generate_answer` 快速链路，跳过相关性评估与查询重写
   - 评估/回答上下文长度改为配置驱动，避免固定超长输入

6. **环境变量示例同步**
   - 更新 `.env`
   - 更新 `.env.example`
   - 新增：`RAG_FAST_MODE`、`RAG_RELEVANCE_MAX_CHARS`、`RAG_ANSWER_MAX_CHARS`
