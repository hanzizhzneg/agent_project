# app.py 导入说明笔记（1-22 行）

本文档解释 `app.py` 顶部导入项分别来自什么库、通用作用是什么、以及它们在本项目中的具体用途。

## 1) Python 标准库

### `from __future__ import annotations`
- **库/来源**：Python 内置特性（future import）
- **通用作用**：推迟类型注解求值，减少前向引用和类型注解兼容问题。
- **本项目作用**：让诸如 `list[Document]`、`tuple[str, ...]` 等类型注解在运行时更稳定，简化函数签名书写。

### `import argparse`
- **库/来源**：Python 标准库
- **通用作用**：构建命令行参数解析器。
- **本项目作用**：支持 `python app.py ingest ...` / `python app.py ask ...` 这类 CLI 用法。

### `import os`
- **库/来源**：Python 标准库
- **通用作用**：读写环境变量、操作系统交互。
- **本项目作用**：读取 `LLM_API_KEY`、`LLM_MODEL`、`LLM_BASE_URL`、`LLM_TEMPERATURE` 等配置。

### `import json`
- **库/来源**：Python 标准库
- **通用作用**：JSON 序列化/反序列化。
- **本项目作用**：在流式问答接口（SSE）里把 token/done 事件打包为 JSON 文本返回前端。

### `from pathlib import Path`
- **库/来源**：Python 标准库
- **通用作用**：跨平台路径对象与文件系统操作。
- **本项目作用**：管理 `knowledge_base`、`vector_db`、`web/index.html` 等目录与文件路径。

### `from typing import Iterator, Literal`
- **库/来源**：Python 标准库（typing）
- **通用作用**：类型提示。
- **本项目作用**：
  - `Iterator`：标注流式输出生成器返回类型；
  - `Literal`：限制 `store_type` 只能是 `"chroma"` 或 `"faiss"`。

## 2) Web API 框架相关

### `from fastapi import FastAPI, HTTPException`
- **库/来源**：`fastapi`
- **通用作用**：构建高性能 Python Web API；`HTTPException` 用于返回规范错误响应。
- **本项目作用**：
  - `FastAPI`：创建服务应用；
  - `HTTPException`：在 `/ingest`、`/ask`、`/ask_stream` 中返回 400/500 错误信息。

### `from fastapi.responses import FileResponse, StreamingResponse`

- **通用作用**：
  - `FileResponse`：直接返回文件；
  - `StreamingResponse`：流式返回数据。
- **本项目作用**：
  - 首页 `/` 返回 `web/index.html`；
  - `/ask_stream` 通过 SSE 实时推送模型输出。

### `from pydantic import BaseModel, Field`
- **库/来源**：`pydantic`
- **通用作用**：数据模型定义、字段校验、自动文档生成。
- **本项目作用**：定义请求/响应模型（如 `IngestRequest`、`AskRequest`、`AskResponse`），约束参数类型和默认值。

## 6) `app.py:25-31` 配置段说明

这段代码本质上是“项目级常量配置区”，提前把**存储类型约束**和**关键路径**集中定义，避免在后续函数里写死字符串，提升可维护性和可读性。

### `StoreType = Literal["chroma", "faiss"]`
- **意义**：限定向量库类型只能是 `chroma` 或 `faiss`。
- **为什么合理**：
  - 给编辑器和类型检查器明确约束，减少传错值；
  - 请求模型和函数签名更清晰。
- **在本项目中的作用**：`/ingest`、`/ask` 等接口会基于该类型选择向量库实现。

### `BASE_DIR = Path(__file__).parent`
- **意义**：把 `app.py` 所在目录作为项目基准路径。
- **为什么合理**：
  - 避免依赖“当前工作目录”导致的路径错乱；
  - 跨终端/跨系统启动更稳定。
- **在本项目中的作用**：所有相对目录都从这里派生，保证 `uvicorn` 从任意位置启动时仍能找到资源。

### `DATA_DIR = BASE_DIR / "knowledge_base"`
- **意义**：知识库源文档目录（PDF/DOCX）统一入口。
- **在本项目中的作用**：`load_documents()` 默认从此目录扫描待入库文件。

### `CHROMA_DIR = BASE_DIR / "vector_db" / "chroma"`
- **意义**：Chroma 本地持久化目录。
- **在本项目中的作用**：`store_type="chroma"` 时向量数据写入/读取该目录。

### `FAISS_DIR = BASE_DIR / "vector_db" / "faiss"`
- **意义**：FAISS 本地索引目录。
- **在本项目中的作用**：`store_type="faiss"` 时保存和加载 FAISS 索引文件。

### `WEB_DIR = BASE_DIR / "web"` 与 `INDEX_HTML = WEB_DIR / "index.html"`
- **意义**：前端静态页面目录与入口文件路径。
- **在本项目中的作用**：`GET /` 通过 `FileResponse` 返回这个页面，形成最简可用的本地前端。

### 小结
- 这段代码是合理的“配置集中化”写法：  
  **统一路径定义 + 类型约束 + 运行时稳定性**。  
  对当前项目来说，它直接决定了文档读入位置、向量库落盘位置和首页入口文件位置。





## 7) `app.py:40-44`（`IngestRequest`）逐行解释

这段代码定义了“入库接口请求体”的数据模型，属于 FastAPI + Pydantic 的标准写法。当前对应接口是 `POST /ingest`。

### `class IngestRequest(BaseModel):`
- **作用**：声明一个请求模型类，继承 `BaseModel` 后可自动做参数校验与类型转换。
- **在本项目中**：`/ingest` 接口接收 JSON 时，FastAPI 会按这个类解析和校验字段。

### `docs_dir: str = Field(default=str(DATA_DIR), description="PDF/Word docs directory")`
- **作用**：
  - 定义字段 `docs_dir`，类型是字符串；
  - 默认值是 `DATA_DIR`（即知识库目录）；
  - `description` 用于 API 文档说明。
- **在本项目中**：
  - 不传时默认扫描 `knowledge_base`；
  - 你也可以传自定义目录，指定要入库的 PDF/DOCX 路径。

### `store_type: StoreType = "chroma"`
- **作用**：
  - 定义向量库类型字段；
  - 类型是 `StoreType`（前面用 `Literal["chroma", "faiss"]` 限定）；
  - 默认 `chroma`。
- **在本项目中**：决定本次入库是写入 Chroma 还是 FAISS。

### `chunk_size: int = 800`
- **作用**：定义切分块大小（每段文本大致字符长度）。
- **在本项目中**：入库时用于 `RecursiveCharacterTextSplitter`，影响检索粒度和上下文覆盖。

### `chunk_overlap: int = 150`
- **作用**：定义相邻文本块重叠长度，减少切分断裂带来的语义丢失。
- **在本项目中**：和 `chunk_size` 一起控制切分策略，提升问答召回连续性。

### 小结
- `IngestRequest` 就是 `/ingest` 的“参数合同”：
  - 约束输入格式；
  - 提供默认值；
  - 直接影响文档读取位置、向量库存储类型和切分效果。
