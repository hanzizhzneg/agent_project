## 2026-04-27 Chroma 持久化兼容修复

### 代码更改
- 文件：`app.py`
- 位置：`build_vector_store(...)` 的 `store_type == "chroma"` 分支
- 变更：
  - 由固定调用 `vectorstore.persist()`
  - 改为仅在对象存在该方法时调用：
    - `if hasattr(vectorstore, "persist"): vectorstore.persist()`

### 变更原因
- 根因：当前 `langchain-chroma` 版本中 `Chroma` 对象不再包含 `persist()` 方法。
- 报错：`Failed to ingest: 'Chroma' object has no attribute 'persist'`
- 目的：兼容新旧版本 API，避免入库失败。

### 影响范围
- 影响：`/ingest`（`chroma` 模式）与 CLI ingest 的持久化流程。
- 不影响：`faiss` 路径及问答接口主流程。
- 风险：低（仅增加兼容判断，不改变核心逻辑）。

### 验证方式
- 启动：`python -m uvicorn app:app --reload`
- 执行：调用 `/ingest`，`store_type=chroma`
- 预期：不再出现 `persist` 属性错误，入库成功。

## 2026-04-27 百炼 API 默认配置调整

### 代码更改
- 文件：`app.py`
- 位置：`get_llm()`
- 变更：
  - 默认模型从 `gpt-4o-mini` 改为 `qwen3.6-35b-a3b`
  - 默认 `LLM_BASE_URL` 增加为 `https://dashscope.aliyuncs.com/compatible-mode/v1`
  - 缺失 `LLM_API_KEY` 的报错提示补充为百炼 Key 语义

- 文件：`README.md`
- 位置：LLM 配置章节
- 变更：
  - 标题与示例改为百炼/Qwen 场景
  - 移除明文示例 Key，改成占位符
  - 补充平台不匹配会触发 `401 invalid_api_key` 的说明

### 变更原因
- 根因：当前项目运行时使用的是百炼模型与百炼 Key，但文档/默认值偏 OpenAI，容易导致鉴权混配报错（401）。
- 目标：让项目“开箱即用”匹配当前使用平台，降低配置错误率。

### 影响范围
- 影响：`LLM_*` 环境变量的默认行为与文档指引。
- 不影响：RAG 业务流程、向量库逻辑、API 路由。

### 验证方式
- 启动：`python -m uvicorn app:app --reload`
- 配置：
  - `LLM_API_KEY=百炼Key`
  - `LLM_MODEL=qwen3.6-35b-a3b`
  - `LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1`
- 预期：`/ask` 或 `/ask_stream` 不再出现 `401 invalid_api_key`（在 Key 正确前提下）。

## 2026-04-27 PDF 解析器优化（附件检索问题）

### 代码更改
- 文件：`app.py`
- 位置：`load_pdf(file_path)`
- 变更：
  - 新增 `PyMuPDF`（`fitz`）解析逻辑作为首选
  - 保留 `PyPDF2` 作为兜底解析器
  - 优先返回 `PyMuPDF` 抽取结果，减少中文 PDF 乱码概率

- 文件：`requirements.txt`
- 变更：
  - 新增依赖：`PyMuPDF`

### 变更原因
- 根因：用户问“附件一”时检索结果常提示无内容，排查发现 PDF 文本抽取乱码，导致召回片段可见但 LLM 难以理解。
- 方案：使用对中文 PDF 抽取更稳定的 `PyMuPDF`，同时保留 `PyPDF2` 兼容兜底。

### 影响范围
- 影响：文档入库阶段的 PDF 文本抽取质量。
- 不影响：向量检索与问答接口结构、数据模型、路由。
- 注意：需重新执行 `/ingest` 让新的抽取结果进入向量库。

### 验证方式
- 安装依赖：`pip install -r requirements.txt`
- 重新建库：`POST /ingest`
- 提问：`附件一的内容是什么`
- 预期：返回内容中可引用“附件一”相关条款，不再持续回复“知识库无法确定”。

## 2026-04-27 修复 FAISS 写入路径报错（Windows 中文路径）

### 代码更改
- 文件：`app.py`
- 变更：
  - `FAISS_DIR` 默认目录从项目内路径改为用户目录下英文路径：
    - `~/enterprise_qa_data/vector_db/faiss`
  - 支持环境变量覆盖：`FAISS_DIR`
  - FAISS 不存在时，报错中包含实际目录，便于排查。

### 变更原因
- 根因：Windows 下 `faiss` 在包含中文字符的路径上可能无法正常读写（`FileIOWriter/FileIOReader` 报 `could not open ...`）。
- 目标：默认使用纯英文路径，避免路径编码兼容问题。

### 影响范围
- 影响：仅 FAISS 索引存储位置。
- 不影响：Chroma 存储、问答接口流程。

### 验证方式
- 点击“构建 FAISS 索引”：应成功写入索引文件。
- 选择 `faiss` 问答：应可正常读取索引并回答。
